# i18n Design: Korean & English Multilingual Support

**Date:** 2026-05-02  
**Status:** Approved

---

## Overview

Add Korean (`ko`) and English (`en`) multilingual support to yujimin.dev. Language is auto-detected from the browser's `accept-language` header and reflected in the URL via locale prefix. No manual language switcher UI.

---

## Scope

**Included:**

- All static UI text (labels, placeholders, button text, empty states, error messages)
- Page metadata (title, description, OpenGraph tags, Twitter card)
- Layout and navigation text (StartMenu, Taskbar, window titles)

**Excluded:**

- Blog post titles and content (MDX files)
- Resume content (fetched from Google Docs)

---

### i18n Config Files

**`src/i18n/routing.ts`**

```ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['en', 'ko'],
  defaultLocale: 'en',
});
```

**`src/i18n/navigation.ts`**

```ts
import { createNavigation } from 'next-intl/navigation';
import { routing } from './routing';

export const { Link, redirect, usePathname, useRouter } =
  createNavigation(routing);
```

**`src/i18n/request.ts`**

```ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = (await requestLocale) ?? routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

## Middleware

Location: `src/middleware.ts`

**Behavior:**

- `/`, `/blog`, `/about-me`, `/resume`, `/guestbook`, `/colophon` тЖТ detect `accept-language` header тЖТ redirect to `/ko/*` or `/en/*`
- `/ko/*`, `/en/*` тЖТ pass through
- `/api/*`, `/_next/*`, static files тЖТ skip middleware entirely

**Language detection logic:**

- `accept-language` contains `ko` тЖТ redirect to `/ko/...`
- All other languages тЖТ redirect to `/en/...` (default)

## Type Safety

`messages/en.json` is used as the base schema. TypeScript infers message key types from this file, providing:

- Key autocomplete in IDE
- Compile-time error on typos or missing keys

Setup in `src/global.d.ts` (or `src/i18n/types.ts`):

```ts
import en from '../messages/en.json';

declare module 'next-intl' {
  interface AppConfig {
    Messages: typeof en;
  }
}
```

## Usage Patterns

**Server components (default):**

```ts
import { getTranslations } from 'next-intl/server';

export default async function GuestbookApp() {
  const t = await getTranslations('guestbook');
  return <h1>{t('title')}</h1>;
}
```

**Client components (interactive only):**

```ts
'use client';
import { useTranslations } from 'next-intl';

export function GuestbookForm() {
  const t = useTranslations('guestbook.form');
  return <input placeholder={t('namePlaceholder')} />;
}
```

**Rule:** Default to `getTranslations()` in server components. Use `useTranslations()` only in `'use client'` components that require interactivity.

## Metadata Localization

Each page uses `generateMetadata` with locale-specific values:

```ts
export async function generateMetadata({ params }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: 'blog' });
  return { title: t('metadata.title') };
}
```

## Testing Strategy

### 1. Key Parity Test (Vitest)

`src/__tests__/i18n/keyParity.test.ts`

Recursively compares key sets of `en.json` and `ko.json`. Fails if any key exists in one but not the other. Enforces that both files stay in sync whenever a new key is added.

### 2. Middleware Routing Tests (Vitest)

- `accept-language: ko-KR` тЖТ redirects to `/ko/*`
- `accept-language: fr-FR` тЖТ redirects to `/en/*` (default)
- `accept-language: en-US` тЖТ redirects to `/en/*`
- `/api/guestbook` тЖТ middleware skipped, no redirect
- `/_next/static/*` тЖТ middleware skipped

### 3. E2E Test Updates (Playwright)

- Update all existing path references from `/blog` тЖТ `/en/blog`, etc.
- Add smoke tests for `/ko` routes (homepage, blog, guestbook)

## Migration Steps (high-level)

1. Install `next-intl`
2. Create `src/i18n/` config files and `messages/en.json` + `messages/ko.json`
3. Update `src/middleware.ts` to use `next-intl` routing
4. Move `src/app/` pages under `src/app/[locale]/`
5. Update `src/app/layout.tsx` (root) and add `src/app/[locale]/layout.tsx`
6. Replace all inline TEXT constants with `getTranslations()` / `useTranslations()` calls
7. Localize metadata in each page
8. Add key parity test
9. Update E2E test paths
